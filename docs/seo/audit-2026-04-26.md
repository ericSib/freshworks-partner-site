## 1 · Synthèse exécutive

Le site Freshworks Partner est en prod, techniquement propre (i18n FR/EN parfait, schema.org de qualité, GA4 consent-gated), mais **sa surface de captage Google est minimale (4 routes indexables) et son funnel de conversion n'est pas instrumenté (0 call-site GA4 réel)**.

Pour atteindre l'objectif PO « Top 10 sur consultant Freshservice à 6 mois » et « conversion >3% », le Sprint 20 cible 7 stories (D28) :

1. Étendre la surface (pages services dédiées Freshservice + Freshdesk FR+EN)
2. Enrichir les signaux Google (schema areaServed UK/BE/CH, OG dynamiques, sitemap stable)
3. Instrumenter le funnel (events GA4 quiz + CTA + contact)
4. Calibrer le ROI affiché (Forrester TEI 2024)

## 2 · On-page SEO

### 2.1 · Title + meta description (état actuel)

| Page | Title | Description | Verdict |
|---|---|---|---|
| `/fr` (home) | « What A Service — Transformez votre investissement Freshworks en avantage compétitif » (95 c.) | 268 c. | Title OK longueur mais brand en tête mange le keyword. Description **trop longue** (Google tronque ~155 c.) |
| `/en` (home) | (idem traduit) | (idem) | Idem |
| `/fr/quiz` | « Service Maturity Index™ — Évaluez votre maturité service — What A Service » | 145 c. | Double tiret moche, keyword manquant |
| `/fr/maturite/itsm/level-1` | « Maturité ITSM Niveau 1 — Mode pompier — What A Service » | 113 c. | Manque « Freshservice » / « consultant » |
| `/fr/mentions-legales` | (hérite layout) | — | Pas critique |

**Action S20** : raccourcir meta description home ≤ 155 chars + reformuler avec keyword principal (US-S20-4).

### 2.2 · Hiérarchie sémantique (homepage)

- **H1** unique : `t("hero.headline")` via `<TextReveal as="h1">` dans `Hero.tsx:59` ✓
- **H2** : un par section (12 sections). Bon ratio.
- **H3** : présents dans Process, Services. Hiérarchie correcte.

### 2.3 · Schema.org (JSON-LD)

**@types détectés en prod** (via `src/config/schema.ts` + `StructuredData.tsx`) :
- Organization (`@id: #organization`, logo, sameAs LinkedIn, geoCircle Paris+500km, contactPoint bilingue)
- Person (Eric Sibomana, 7 credentials dont ITIL 4 / PRINCE2 / DORA / PSM / PSPO / Freshworks Advanced)
- Service × N (généré dynamiquement depuis `OFFER_SLUGS`)
- FAQPage (6 Q/A intent-aligned : « Qu'est-ce qu'un consultant Freshworks », « Combien coûte une implémentation Freshservice », « Comment migrer de ServiceNow vers Freshservice », « Freshservice vs ServiceNow »)
- BreadcrumbList (sur quiz + maturité)
- Service spécifique sur pages maturité (`#maturity-itsm-level-1`)

**Gaps identifiés** :
- `areaServed = "France"` exclusivement → bloque l'intent UK/BE/CH déclaré dans le brief PO. **Action S20** : étendre à 4 pays (US-S20-4).
- Pas de schema `Offer` avec prix « à partir de » (D15 acté) — manque côté Rich Results.
- Pas de `Review` / `AggregateRating` (cohérent — pas encore de témoignages structurés).

### 2.4 · Open Graph + Twitter Cards

- Homepage : OG complet (title/description/url/siteName/locale/type/image 1200×630) + Twitter `summary_large_image`. Image unique statique `/images/og-default.png`.
- **Quiz, maturité, mentions-légales** : `generateMetadata` ne re-déclare PAS `openGraph` ni `twitter` → ces pages **héritent de l'OG du root**. **Bug réel** : un partage du quiz sur LinkedIn affichera l'OG du root, pas du quiz.

**Action S20** : OG image dynamique per route via Next.js 16 `opengraph-image.tsx` (US-S20-3).

### 2.5 · Canonical + hreflang

- `/fr` : canonical `https://freshworks.whataservice.fr/fr` + alternates `fr`, `en`, `x-default=fr` ✓
- `/fr/quiz` et `/fr/maturite/...` : canonical + alternates `fr`/`en` mais **pas de `x-default`** (oubli `[locale]/quiz/page.tsx:18-23` et `[locale]/maturite/[segment]/[level]/page.tsx:38-44`).

**Action future** : compléter `x-default` sur toutes les routes (absorbé dans US-S20-2 quand on touche les nouvelles pages services).

### 2.6 · Internal linking — faible

- `NAV_LINKS` (`src/config/navigation.ts:13`) ne contient que des **ancres** — aucun lien inter-pages depuis la nav.
- Pages maturité : pas de « voir les autres niveaux » (légitime, ESM + niveaux 2-5 = backlog Phase 2 D31).
- **Aucun lien sortant** vers Freshworks.com officiel (autorité de domaine adjacente — bénéfice réciproque potentiel via partenariat).

**Action S20** : enrichir Header NAV_LINKS + Footer + cross-link Freshservice ↔ Freshdesk dans US-S20-2.

## 3 · Technical SEO

### 3.1 · Sitemap (`src/app/sitemap.ts`)

**Avant US-S20-1** :
- 5 routes × 2 langues = 10 URLs total
- `lastModified: new Date()` à chaque build → tous les `lastmod` changent à chaque deploy même si le contenu n'a pas bougé. **Bruit pour les crawlers.**

**Après US-S20-1** (livré ce jour) :
- `lastModified` figé par route via `LAST_MODIFIED` map (ISO date par contenu)
- Test `src/app/__tests__/sitemap.test.ts` assert : 5 entries, alternates bilingues, lastModified deterministe + dans le passé strict, priorités correctes
- 4/4 tests pass

**Reste à faire S20+** : ajouter les 2 nouvelles routes services Freshservice + Freshdesk (US-S20-2) → sitemap passera à 14 URLs.

### 3.2 · Robots (`src/app/robots.ts`)

- Allow `/`, disallow `/api/` `/_next/` ✓
- **GEO-friendly** : 5 user-agents IA explicitement autorisés (GPTBot, ChatGPT-User, PerplexityBot, ClaudeBot, Google-Extended). Bonne pratique 2026.
- Sitemap déclaré.

### 3.3 · Indexabilité

- Pas de `<meta name="robots" content="noindex">` détecté → toutes les routes Next.js sont indexables ✓.
- HTTPS via Vercel + Let's Encrypt R12.

### 3.4 · Core Web Vitals

**Non mesurés ici** (WebFetch et PageSpeed bloqués dans la session). À lancer manuellement post-déploiement S20 :

```
https://pagespeed.web.dev/?url=https://freshworks.whataservice.fr/fr
https://pagespeed.web.dev/?url=https://freshworks.whataservice.fr/fr/quiz
```

**Risques connus** :
- Hero `<Image priority>` 800×900 + carousel logos client + animations TextReveal → potentiel impact LCP/CLS
- `Hero.tsx` est `"use client"` complet — refactor possible : shell server, animation enfant client (à backlogger en S21+ via Refactoring Radar)

## 6 · Conversion (depuis le code)

### 6.1 · Funnel apparent

- Hero CTA primaire `#contact` (scroll-to-form, 1 clic)
- Hero CTA secondaire `/quiz` (1 clic, parcours quiz multi-écrans)
- Sticky CTA banner global (`StickyCtaBanner`)
- Cookie banner RGPD-gated avant tracking

### 6.2 · Tracking GA4 — **0 call-site réel**

`grep -rn "trackQuizComplete\|trackContactSubmit\|trackEvent" src/` retourne **rien** hors `analytics.ts` et `__tests__`. Les helpers sont **morts**.

**Conséquence** : on ne mesure ni le funnel quiz, ni les CTA, ni les conversions formulaire. Impossible d'optimiser sans data.

**Action S20** : instrumentation complète (US-S20-5 quiz + US-S20-6 CTA/contact).

### 6.3 · Friction visible (à confirmer en preview live)

- StickyCtaBanner peut masquer le formulaire sur mobile (à tester 375px)
- CookieBanner sur first paint = peut compromettre LCP perçu
