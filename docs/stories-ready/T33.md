## User Story

```
En tant que developpeur ecrivant des tests Vitest qui touchent localStorage,
je veux un mock localStorage centralise dans vitest.setup.ts (avec reset automatique
entre fichiers de test),
afin d'eviter le pattern Object.defineProperty(global, "localStorage", ...) qui
leak entre fichiers et casse `vi.clearAllMocks()` (Drop S20 D11).
```

---

## Contexte

Pendant le Sprint 20, le mock localStorage local de `src/lib/__tests__/analytics.test.ts:35` (`Object.defineProperty(global, "localStorage", { value: localStorageMock })`) a leak vers `CookieBanner.test.tsx`. Le `vi.clearAllMocks()` du second fichier a clear les `vi.fn()` du store mock, rendant `localStorage.setItem` non-fonction → tous les tests CookieBanner ont fail au 1er run. 10 minutes de debug + workaround mock localStorage local dans CookieBanner.

Solution : extraire le mock dans un setupFile partage avec reset automatique en `beforeEach`.

## Notes techniques / decisions

### Pattern recommande (vitest.setup.ts)

```typescript
import { vi, beforeEach } from "vitest";

const createLocalStorageMock = () => {
  const store: Record<string, string> = {};
  return {
    getItem: vi.fn((key: string) => store[key] ?? null),
    setItem: vi.fn((key: string, value: string) => { store[key] = value; }),
    removeItem: vi.fn((key: string) => { delete store[key]; }),
    clear: vi.fn(() => { Object.keys(store).forEach(k => delete store[k]); }),
    key: vi.fn((index: number) => Object.keys(store)[index] ?? null),
    get length() { return Object.keys(store).length; },
  };
};

beforeEach(() => {
  Object.defineProperty(globalThis, "localStorage", {
    value: createLocalStorageMock(),
    writable: true,
    configurable: true,
  });
});
```

Cle : `createLocalStorageMock()` est appelee a CHAQUE `beforeEach()` → store frais + nouveaux `vi.fn()` → pas de leak entre fichiers.
