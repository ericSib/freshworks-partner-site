## Contexte

Le module `src/lib/analytics.ts` (livre en Sprint 12 via US-22.8) est actuellement couvert a **37.5% seulement** :

| Fonction | Couverte ? |
|---|---|
| `hasConsent()` | ✅ |
| `setConsent()` | ✅ |
| `trackEvent()` | ❌ |
| `trackQuizComplete()` | ❌ |
| `trackContactSubmit()` | ❌ |

**Risque silencieux** : si un tracker casse (rename d'event, typo payload, changement de categorie), aucun signal ne remonte — perte de donnees funnel conversion pendant des semaines avant detection manuelle via GA4 dashboard. Or, ces trackers sont au coeur du KPI *"Conversion visiteur → lead (quiz) : 15-30%"* du Product Goal.

## Criteres d'acceptation (Gherkin)

### Scenario 1 : trackEvent bloque sans consentement

```gherkin
Etant donne hasConsent() retourne false (localStorage non defini)
Et window.gtag est mocke
Quand trackEvent({action:"test_action"}) est invoque
Alors window.gtag n'est PAS appelee
```

### Scenario 2 : trackEvent graceful si gtag absent

```gherkin
Etant donne consent="granted" dans localStorage
Et window.gtag est undefined
Quand trackEvent({action:"test"}) est invoque
Alors aucune exception n'est levee
```

### Scenario 3 : trackEvent avec payload complet

```gherkin
Etant donne consent="granted" et window.gtag mocke
Quand trackEvent({action:"click", category:"nav", label:"header", value:1}) est invoque
Alors gtag est appele avec "event", "click", {event_category:"nav", event_label:"header", value:1}
```

### Scenario 4 : trackQuizComplete formate le payload correctement

```gherkin
Etant donne consent="granted" et window.gtag mocke
Quand trackQuizComplete("itsm", 72.4) est invoque
Alors gtag est appele avec "event", "quiz_complete", {event_category:"engagement", event_label:"itsm", value:72}
```

### Scenario 5 : trackContactSubmit emet l'event conversion

```gherkin
Etant donne consent="granted" et window.gtag mocke
Quand trackContactSubmit() est invoque
Alors gtag est appele avec "event", "contact_submit", un objet contenant event_category:"conversion"
```
