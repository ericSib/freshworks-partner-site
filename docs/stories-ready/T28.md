## User Story

```
En tant que developpeur ou QA travaillant sur une story SEO,
je veux que la DoD impose explicitement de lister les tests E2E qui asserent
sur les meta/title impactes par la story,
afin que la modification d'un texte SEO ne casse pas silencieusement les tests
existants (regression D40 vecue post-Sprint 20 sur US-S20-1).
```

---

## Contexte

Le commit `b41ab07` (test e2e: align homepage title assertions with SEO-optimized US-S20-1) a corrige post-coup une regression : US-S20-1 avait change le `<title>` dans `src/messages/{fr,en}.json` mais n'avait pas mis a jour les assertions `toHaveTitle(/.../)` dans `tests/e2e/homepage.spec.ts`. Cela a ete masque par le blocage CI lockfile drift, mais aurait du etre detecte pendant la story elle-meme.

D40 acte que ce n'est pas un Drop S20 distinct, c'est un **bug coverage** de US-S20-1 qui appelle un Try precis sur la DoD.

## Notes techniques / decisions

### Forme : checklist conditionnelle (et non systematique)

La regle s'applique uniquement aux stories qui touchent du SEO/meta. Forcer cette checklist sur toutes les stories serait du gaspillage cognitif. La condition explicite "SI la story modifie titre/meta/h1/og-title" rend la regle ciblee.

### Pas de test code

C'est une regle process. Le seul "test" est l'application reussie sur les 3 stories US-S21 du sprint. Si l'une oublie d'appliquer la nouvelle DoD, la PO Sprint Review le detectera (pattern T39 = test live incognito).
