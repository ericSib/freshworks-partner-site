## User Story

```
En tant que developpeur maintenant le module PDF du quiz,
je veux que `generate-pdf.ts` soit decompose en 4 helpers (header / scores / quick-wins / footer),
afin de pouvoir tester et faire evoluer chaque section du PDF independamment.
```

## Notes techniques / decisions

### Pattern Fowler applique

**Extract Method** simple — pas de changement architectural, pas de nouvelle abstraction.

### Risque cle : effet de bord jsPDF

jsPDF mute son etat interne (curseur Y, page courante). Chaque helper doit :
- Soit consommer le doc et avancer le curseur via les API jsPDF natives
- Soit retourner explicitement la nouvelle position Y (option moins ergonomique mais plus pure)

**Decision** : approche par mutation (idiomatique jsPDF) — le doc est passe par reference, chaque helper le mute.

### Pourquoi pas Sprint 19

Effort 2 pts > 4 pts buffer disponible apres injection des 3 quick wins. Reportage S20 acte au refinement Radar §3.
