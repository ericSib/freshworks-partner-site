## User Story

```
En tant que developpeur faisant evoluer le formulaire de contact ou ajoutant un nouveau formulaire,
je veux que la className des inputs soit centralisee dans un composant `FormInput`,
afin que toute evolution du style (focus ring, padding, border) ne necessite qu'une seule modification.
```

## Notes techniques / decisions

### Pattern Fowler applique

**Extract Function/Component** classique. Le composant vit dans `src/components/ui/` (deja le home des composants UI atomiques type `SectionTag`, `SectionHeader`).

### Signature proposee

```typescript
type FormInputProps = React.InputHTMLAttributes<HTMLInputElement> & {
  label: string;
  error?: string;
};

export function FormInput({ label, error, id, ...rest }: FormInputProps) {
  const errorId = error && id ? `${id}-error` : undefined;
  return (
    <div>
      <label htmlFor={id}>{label}</label>
      <input
        id={id}
        aria-describedby={errorId}
        aria-invalid={!!error}
        className="w-full bg-white/[0.03] border border-white/8 rounded-lg px-4 py-3.5 text-surface placeholder:text-slate-600 focus:outline-none focus:border-accent/40 focus:shadow-[0_0_0_3px_rgba(184,146,106,0.08)] transition-all"
        {...rest}
      />
      {error && <p id={errorId} role="alert">{error}</p>}
    </div>
  );
}
```

### Pourquoi pas un composant plus generique (FormField, etc.)

YAGNI (Manifeste P10). 3 call sites identiques aujourd'hui. Si un nouveau formulaire arrive (newsletter, CTA inline), on enrichit alors — pas avant.
