## User Story

```
En tant que developpeur,
je veux decouper QuizResultsPreview.tsx en 4 sous-composants a responsabilite unique,
afin de faciliter les futures iterations (A/B test email gate, variations CTA par segment).
```

---

## Contexte

`src/components/quiz/QuizResultsPreview.tsx` fait **261 lignes** et melange **4 responsabilites distinctes** :

1. **Score header** : score + badge urgency + description niveau
2. **Radar chart wrapper** : composant deja extrait mais integre ici
3. **Quick wins** : loop sur weakestDimensions avec angles commerciaux
4. **Email gate + breakdown gated + bouton PDF** : form + detailed dimension breakdown + generateQuizPdf

**Code smell Fowler** : Large Class (>200 lignes, >3 responsabilites) — voir skill dev-refactoring, catalog.

**Impact strategique** : les stories futures US-18.10+ (A/B testing email gate, variations CTA par niveau de maturite, tracking granulaire par zone) sont **bloquees tant que le composant est monolithique**. Le cout de refacto augmente lineairement avec chaque nouvelle feature ajoutee au fichier.

**Bonus** : elimine l'import mort `Link` (finding #2 du quality gate) dans le meme passage.

- ❌ Changer la signature des data-testid utilises par l'E2E
